# Professional services

`sap-abap-mcp` is free and open-source software. Paid services are available
for organizations that want help evaluating or adopting it in a controlled SAP
DEV/QAS environment.

The service does not require production write access. SAP profiles,
credentials, source code, and tool results remain on the customer's machine.

## Public evidence

- The current public release exposes 53 MCP tools for SAP ABAP workflows.
- The TypeScript build and all 119 automated contract and safety tests pass as
  of July 20, 2026.
- The implementation, test suite, release history, and safety controls can be
  reviewed in the [public repository](https://github.com/Coaspe/sap-abap-mcp)
  before a paid engagement begins.

These checks validate the local software. They do not replace acceptance
testing against the customer's authorized SAP release and configuration.

## Compatibility assessment

A short assessment to determine whether a pilot can run safely.

- Review the SAP release, ADT endpoint, authentication method, and intended
  development workflow.
- Verify a dedicated DEV/QAS account and read-only connection where access is
  available.
- Identify unsupported prerequisites before a larger engagement starts.
- Deliver a concise compatibility and next-step summary.

Starting range: KRW 350,000-500,000 or USD 300-400. Taxes, travel, and unusual
access requirements are quoted separately.

## Five-day DEV/QAS pilot

A fixed-scope evaluation with documented acceptance evidence.

1. Verify the connection, credential handling, and environment classification.
2. Test agreed source inspection and diagnostic workflows.
3. Run approved ABAP Unit or ATC checks where the system supports them.
4. Demonstrate one controlled change in an explicitly allowed package, or
   document why customer policy prevents it.
5. Deliver an acceptance report covering successful, failed, and unverified
   capabilities, along with operating safeguards and next steps.

Starting range: KRW 1,800,000-3,000,000 or USD 1,500-2,500 for one SAP
DEV/QAS connection. The final quote depends on access, meeting time, required
documentation, and the agreed acceptance criteria.

## Boundaries

- No production changes.
- No unrestricted data extraction.
- No credential sharing through chat, email, or public GitHub issues.
- No penetration testing, endpoint bypasses, or activity outside the customer's
  written authorization.
- SAP-dependent behavior is reported as unverified until it succeeds against
  the customer's authorized system.
